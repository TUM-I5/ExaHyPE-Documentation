Detailed Description

Simple matrix generator assuming that we use a Gauss Lobatto unknown layout.

If you use this class, you assuem that a cell hosts \( (p+1)^d \) nodes carrying Legendre polynomials. As you use a Gauss Lobatto layout, the "outer" nodes will end up on the cell faces. Each node can hold a fixed number of unknowns. If you multiply the number of nodes per cell with the corresponding number of unknowns, you get the degrees of freedom within a global assembly matrix which correspond to volumetric data.

mghype.api.matrixgenerators.GaussLobatto.GaussLobatto.get_averaging_riemann_solver ( self)

Simple averaging Riemann solver.

We know that this solver is extremely diffusive, but it makes sense to provide this one as a simple debugging mechanism. In line with our DG description, it is clear that the dimension of the result matrix is \( dof \times 2dof \) where dof is the number of degrees of freedom projected onto a face.

Please note that unknowns_per_face_node= \( 3 \cdot dof \) in the equation above.
