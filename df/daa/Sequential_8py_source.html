24def translate_one_step_of_species_into_mesh_traversal(
25 step,
26 current_species_set,
27 mesh_traversals,
28 alter_particle_position,
29 use_continuous_particles_sequences,
30 step_type: StepType,
31 verbose,
32):
33 """!
34
35 Translate one step of species into mesh traversal
36
37 Takes one step of the species and adds it to the list of steps. That is,
38 you get one more entry within mesh_traversals. Further to that, the
39 routine also appends a boolean arker to alter_particle_position. It
40 shows if the mesh will change the particle position (or search radius)
41 or not.
42
43 This routine is a helper routine of map_initialisation_steps_onto_separate_mesh_traversals()
44 for example. Note that the returned steps are not fully functionally.
45 If a step alters the particles within the mesh, you still have to resort
46 the particles. That's not done in the step added by this routine, i.e.
47 these steps lack the actual particle sorting. Any sorting has to spread
48 over two steps, so it makes no sense to add it here. Instead, we befill
49 the alter_particle_position, and we expect the calling routine to
50 supplement the actual particle sorting later on in a postprocessing
51 step once all steps are assembled.
52
53
54 @param current_species_set: Particle
55 You get a specific algorithmic step as input. The current_species_set
56 highlights to which this step originally belonged to.
57
58 @param mesh_traversals: [peano4.solversteps.Step]
59 Inout parameter. We append a new step to this sequence.
60
61 @param alter_particle_position: [Bool]
62 Inout parameter. We append a new marker to this sequence.
63
64 """
65 new_step = peano4.solversteps.Step(
66 "{}{}{}_sweep{}".format(
67 get_observer_prefix(step_type),
68 current_species_set.particle_model.name,
69 step.name,
70 len(mesh_traversals),
71 ),
72 add_user_defined_actions=False,
73 )
74
75 action_set_update_marker = UpdateParticleMarker(particle_set = current_species_set,
76 automatically_set_marker_in_leave_cell=True,
77 make_reset_separate_task=False,
78 )
79
80 action_set_update_marker.descend_invocation_order = (
81 new_step.highest_descend_invocation_order() + 1
82 )
83 action_set_update_marker.parallel = True
84 new_step.add_action_set(action_set_update_marker)
85
86 #
87 # Pick computational kernel
88 # =========================
89 # This is where the actual computation per step is done. We have two
90 # fundamentally different kernels: one keeps track of (multiscale)
91 # neighbours, while the other just updates the particles within the
92 # octant (cell).
93 #
94 if step.dependencies == AlgorithmStep.Dependencies.NEIGHBOURS:
95 if use_continuous_particles_sequences:
96 if verbose:
97 print(
98 "GRAPH COMPILER DEBUG [translate_one_step_of_species_into_mesh_traversal]: step no {} ({}) on particle {} can use the peano4.toolbox.particles.UpdateParticle_MultiLevelInteraction_StackOfLists_ContiguousParticles action set and hence optimise aggressively".format(
99 len(mesh_traversals),
100 step.name,
101 current_species_set.particle_model.name,
102 )
103 )
104 action_set_particle_particle_interaction = peano4.toolbox.particles.UpdateParticle_MultiLevelInteraction_StackOfLists_ContiguousParticles(
105 particle_set=current_species_set,
106 particle_particle_interaction_kernel=step.cell_kernel,
107 touch_vertex_first_time_compute_particle_update_kernel=step.touch_vertex_first_time_kernel,
108 touch_vertex_last_time_compute_particle_update_kernel=step.touch_vertex_last_time_kernel,
109 prepare_traversal_kernel=step.prepare_traversal_kernel,
110 unprepare_traversal_kernel=step.unprepare_traversal_kernel,
111 additional_includes=step.includes,
112 active_particle_set=step.input_particles,
113 )
114 else:
115 if verbose:
116 print(
117 "GRAPH COMPILER DEBUG [translate_one_step_of_species_into_mesh_traversal]: step no {} ({}) on particle {} uses the peano4.toolbox.particles.UpdateParticle_MultiLevelInteraction_Sets".format(
118 len(mesh_traversals),
119 step.name,
120 current_species_set.particle_model.name,
121 )
122 )
123 action_set_particle_particle_interaction = peano4.toolbox.particles.UpdateParticle_MultiLevelInteraction_Sets(
124 particle_set=current_species_set,
125 particle_particle_interaction_kernel=step.cell_kernel,
126 touch_vertex_first_time_compute_particle_update_kernel=step.touch_vertex_first_time_kernel,
127 touch_vertex_last_time_compute_particle_update_kernel=step.touch_vertex_last_time_kernel,
128 prepare_traversal_kernel=step.prepare_traversal_kernel,
129 unprepare_traversal_kernel=step.unprepare_traversal_kernel,
130 additional_includes=step.includes,
131 active_particle_set=step.input_particles,
132 )
133 action_set_particle_particle_interaction.descend_invocation_order = (
134 new_step.highest_descend_invocation_order() + 1
135 )
136 action_set_particle_particle_interaction.parallel = True
137 new_step.add_action_set(action_set_particle_particle_interaction)
138 elif step.dependencies == AlgorithmStep.Dependencies.SELF:
139 if use_continuous_particles_sequences:
140 if verbose:
141 print(
142 "GRAPH COMPILER DEBUG [translate_one_step_of_species_into_mesh_traversal]: step no {} ({}) on particle {} uses peano4.toolbox.particles.UpdateParticle_SingleLevelInteraction_ContiguousParticles action set".format(
143 len(mesh_traversals),
144 step.name,
145 current_species_set.particle_model.name,
146 )
147 )
148
149 action_set_particle_update = peano4.toolbox.particles.UpdateParticle_SingleLevelInteraction_ContiguousParticles(
150 particle_set=current_species_set,
151 particle_cell_update_kernel=step.cell_kernel,
152 touch_vertex_first_time_compute_particle_update_kernel=step.touch_vertex_first_time_kernel,
153 touch_vertex_last_time_compute_particle_update_kernel=step.touch_vertex_last_time_kernel,
154 prepare_traversal_kernel=step.prepare_traversal_kernel,
155 unprepare_traversal_kernel=step.unprepare_traversal_kernel,
156 additional_includes=step.includes,
157 active_particle_set=step.input_particles,
158 )
159 action_set_particle_update.descend_invocation_order = (
160 new_step.highest_descend_invocation_order() + 1
161 )
162 action_set_particle_update.parallel = True
163 new_step.add_action_set(action_set_particle_update)
164 else:
165 if verbose:
166 print(
167 "GRAPH COMPILER DEBUG [translate_one_step_of_species_into_mesh_traversal]: step no {} ({}) on particle {} uses peano4.toolbox.particles.UpdateParticle_SingleLevelInteraction action set".format(
168 len(mesh_traversals),
169 step.name,
170 current_species_set.particle_model.name,
171 )
172 )
173
174 action_set_particle_update = peano4.toolbox.particles.UpdateParticle_SingleLevelInteraction(
175 particle_set=current_species_set,
176 particle_cell_update_kernel=step.cell_kernel,
177 touch_vertex_first_time_compute_particle_update_kernel=step.touch_vertex_first_time_kernel,
178 touch_vertex_last_time_compute_particle_update_kernel=step.touch_vertex_last_time_kernel,
179 prepare_traversal_kernel=step.prepare_traversal_kernel,
180 unprepare_traversal_kernel=step.unprepare_traversal_kernel,
181 additional_includes=step.includes,
182 active_particle_set=step.input_particles,
183 )
184 action_set_particle_update.descend_invocation_order = (
185 new_step.highest_descend_invocation_order() + 1
186 )
187 action_set_particle_update.parallel = True
188 new_step.add_action_set(action_set_particle_update)
189 else:
190 assert False, "step {} seems to have no action at all".format(step)
191
192 #
193 # As we have a 1:1 mapping, we can directly translate the effect into flags
194 #
195 if (
196 step.effect== AlgorithmStep.Effect.CHANGE_POSITION_OR_INTERACTION_RADIUS_AND_MIGHT_RERUN
197 or
198 step.effect == AlgorithmStep.Effect.CHANGE_POSITION_OR_INTERACTION_RADIUS
199 ):
200 alter_particle_position.append(True)
201 else:
202 alter_particle_position.append(False)
203
204 mesh_traversals.append(new_step)

261 and alter_particle_position[-1]

264 add_additional_dummy_sweep = AddAdditionalDummySweep.BEFORE
265 elif (
266 particle_sorting_and_storage == ParticleSortingAndStorage.SORT_WITH_LIFTS_AND_DROPS_COALESCED_PARTICLE_STORAGE

302 )

305 if verbose:
306 print(
307 "GRAPH COMPILER DEBUG [map_onto_separate_mesh_traversals]: step no {} ({}) on particle {} has to be dummy step to clean up sorting. Will not alter the particle position itself".format(
308 len(mesh_traversals),
309 current_step_of_species.name,
310 current_species_set.particle_model.name,
311 )
312 )
313 new_step = peano4.solversteps.Step(
314 "DummyStep{}{}_sweep{}".format(
315 current_species_set.particle_model.name,
316 current_step_of_species.name,
317 len(mesh_traversals),
318 ),
319 add_user_defined_actions=False,
320 )
321 new_step.add_action_set(swift2.actionsets.DummyStep())
322 mesh_traversals.append(new_step)
323 alter_particle_position.append(False)

325 if step_type == StepType.INITIALISATION and len(alter_particle_position) > 0:
326 # When we insert particles initially, they will be scattered. So it is

330 # compiler does not distinguish initialisation from time stepping and

334 alter_particle_position[0] = True
335 alter_particle_position[-1] = True
336
337
338 if (
339 particle_sorting_and_storage==ParticleSortingAndStorage.SORT_WITH_LIFTS_AND_DROPS_SCATTERED_PARTICLE_STORAGE
340 or
341 particle_sorting_and_storage==ParticleSortingAndStorage.SORT_WITH_LIFTS_AND_DROPS_COALESCED_PARTICLE_STORAGE
342 ):
343 sort_with_lifts_and_drops = True
344 elif (
345 particle_sorting_and_storage==ParticleSortingAndStorage.SORT_ON_THE_FLY_SCATTERED_PARTICLE_STORAGE
346 or
347 particle_sorting_and_storage==ParticleSortingAndStorage.SORT_ON_THE_FLY_COALESCED_PARTICLE_STORAGE
348 ):
349 sort_with_lifts_and_drops = False
350 else:
351 assert False, "variant {} not implemented".format(particle_sorting_and_storage)
352
353
354 add_dynamic_mesh_refinement_and_particle_resorting(
355 mesh_traversals,
356 alter_particle_position,
357 get_species_set(sequence_of_steps),
358 sort_with_lifts_and_drops=sort_with_lifts_and_drops,
359 verbose=verbose,
360 )
361
362 if verbose:
363 print(
364 "GRAPH COMPILER DEBUG [map_onto_separate_mesh_traversals]: finished graph construction"
365 )
366
367 return mesh_traversals

371def map_onto_separate_mesh_traversals(
372 sequence_of_steps,
373 particle_sorting_and_storage: ParticleSortingAndStorage,
374 step_type: StepType,
375 verbose,
376):
377 """!
378
379 Simpest graph compiler mapping each step per particle onto one mesh traversal
380
381 Very simple compiler which takes each individual algorithmic step
382 per species and maps it onto one mesh traversal. We won't get any
383 fancy parallelism besides the core domain decomposition.
384
385 As we work with a strictly synchronised way, we can ignore in and
386 outgoing dependencies which affect the global state. They are
387 implicitly handled.
388
389 It is important to recognise the inter-dependencies between steps:
390
391 We do not have to care about the plotting: It always resorts the
392 particles, and it always re-analyses which particles are local for
393 a tree. Consequently, the plotting will throw away local particle
394 copies if particles have meanwhile left the domain. We may assume
395 that all particles are in the right place after the plotting. We
396 also may assume that the initialisation inserts particles only at
397 the right point. The realisation of this behaviour can be found in
398 Project.generate_Peano4_project().
399
400 Due to the observations above, we can conclude that particles might
401 have moved in the first step of the algorithmic sequence if and
402 only if the last step has changed a position. Otherwise, we know
403 that all is in place. As we run through the species one by one, we
404 also have to make this reasoning one by one.
405
406 ## Initialisation
407
408 The initialisation is no different to the time stepping besides two
409 places:
410
411 1. We have to use the initialisation steps rather than the
412 algorithm steps.
413 2. We have to insert an additional dummy sweep initially. After
414 users have inserted their particles, they are scattered all
415 over the place. If the first initialisation phase then relies
416 on continuous AoS data, it will crash.
417
418 The latter "tweak" obviously can be ignored whenever we work without
419 coalesced memory.
420
421 ## Tree statistics
422
423 We assume that there is a tree marker already. So our analysis
424 will create it, but we assume that it is already out there, so
425 we don't have to add it actually to the project.
426
427 ## Adaptive mesh refinement
428
429 If the mesh changes, we have to resort. In the second step of the
430 sorting, particles might still be dropped. Throughout drops, we may
431 not evaluate particle-particle interactions, as the mesh-particle
432 topology continues to change, so we'll evaluate stuff twice. Therefore,
433 we have to insert dummy steps. This is not a problem when we bucket
434 sort.
435
436
437 @param sequence_of_steps: [(Species,Step)]
438 Sequence of tuples each consisting of one species and one step. Typically
439 constructed through functions within SpeciesStepOrdering.py.
440
441 @param particle_sorting: ParticleSorting
442
443 @param observer_preamble: String
444 Usually people pass in "Init" or "Step" here to distinguish observers for
445 the initialisation from the actual particle update steps.
446
447 @param verbose: {True, False}
448
449 @return mesh_traversals
450
451 """
452 return map_onto_separate_mesh_traversals_with_generic_mapper(
453 sequence_of_steps,
454 particle_sorting_and_storage,
455 step_type,
456 verbose,
457 translate_one_step_of_species_into_mesh_traversal
458 )
