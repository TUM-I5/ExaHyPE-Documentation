6class ShrinkPatches(Filter):
7 """
8 Extract the fine grid information
9
10 This is a very expensive filter if it is applied on large patch
11 sets, as it has to compare each patch with each other patch.
12 If you want to use it in an economically sensible way, I
13 recommend that you first apply other, cheaper filters to bring
14 the patch count down.
15
16 Even though the overlap operation is idempotent - if a patch
17 is not a fine grid patch within a small local dataset written
18 by one tree, then it can't be part of the fine grid in the
19 overall data set - I found that it is usually faster to
20 concatenate all data and then to apply the filter.
21 """
