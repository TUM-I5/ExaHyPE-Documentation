Detailed Description

DG solver adding tasks to baseline code.

See the documentation for solvers.api.DiscontinuousGalerkinDiscretisationPointWiseRiemannSolver.DiscontinuousGalerkinDiscretisationPointWiseRiemannSolver first. The class adds a few simple features:

  1. We create a cell marker which holds the task numbers.
  2. This one is added to the mesh within add_to_Peano4_datamodel().
  3. We tell both the mesh initialisation and each solve step to use the data.
  4. We swap out the residual computation and the cell update with a bespoke task-based implementation

This is the \( \omega _c \) from the description above and typically chosen form \( ]0,1[ \) though some scenarios benefit from a slight overrelaxation.
assembly_matrix\( \mathbb{R}^{K^c * (p+1)^{d} \times K^c * (p+1)^{d}}\) This is the cell-to-cell matrix which defines how the cell nodes couple to one another. Is is a square matrix. You can pass in None. In this case, the generated C++ code will not compile, but you will be allowed to fill it in manually.
assembly_matrix_scaling\( \mathbb{R}^{K^c * (p+1)^{d} \times K^c * (p+1)^{d}}\) This is a matrix of ints that prescribes how we scale each element in the assembly_matrix ie for each i,j, we do: assembly_matrix[i,j] *= ( h )^( assembly_matrix_scaling[i,j] )
mass_matrix\( \mathbb{R}^{K^c * (p+1)^{d} \times K^c * (p+1)^{d}}\)
mass_matrix_scaling\( \mathbb{R}^{K^c * (p+1)^{d} \times K^c * (p+1)^{d}}\) This is a matrix of ints that prescribes how we scale each element in the mass_matrix ie for each i,j, we do: mass_matrix[i,j] *= ( h )^( mass_matrix_scaling[i,j] )
face_from_cell\( \mathbb{R}^{2d * 2K^f_{f \gets c} * (p+1)^{d-1} \times K^c * (p+1)^{d}}\) This matrix projects the cell solution onto all of the faces. Note the extra factor of \( 2d \). The structure of this matrix is quite important. The rows should be structured in blocks of \( 2K^f_{f \gets c} * (p+1)^{d-1} \), ie one for each face, in the familiar Peano ordering. We note that this is the same as the number of projections we store per face. We note that, since the 0th face has left-side projections which should be set outside of the cell, that at least half of the rows of the matrix should be empty. We also note that each face contains multiple nodes, and they should be ordered sequentially in the typical Peano fashion, i.e. greedily along each coordinate axis until full, starting from the negative. For each face, with two nodes, with two values (q and u) being projected onto the face, the rows should be ordered as follows:
  1. Negative projection of q onto node 0
  2. Negative projection of q onto node 1
  3. Negative projection of u onto node 0
  4. Negative projection of u onto node 1
  5. Positive projection of q onto node 0
  6. Positive projection of q onto node 1
  7. Positive projection of u onto node 0
  8. Positive projection of u onto node 1 In two dimensions, all of the negative projections should be 0 on faces 0 and 1. Ditto for the positive projections on faces 2 and 3.
cell_from_face\( \mathbb{R}^{K^c * (p+1)^{d} \times 2d K^f * (p+1)^{d-1}}\) This matrix projects the face solution onto the cells. Note the extra factor of \( 2d \), similar to face_from_cell matrix. Some clever indexing is required in the backend here. The enumeration of the cols should follow a similar pattern to the rows of face_from_cell.
cell_from_face_scaling\( \mathbb{R}^{K^c * (p+1)^{d} \times 2d K^f * (p+1)^{d-1}}\) This is a matrix of ints that prescribes how we scale each element in the cell_from_face ie for each i,j, we do: cell_from_face[i,j] *= ( h )^( cell_from_face_scaling[i,j] )
Riemann_matrix\( \mathbb{R}^{ K^f (p+1)^{d-1} \times 2K^f_{f \gets c} (p+1)^{d-1}} \) The matrix that combines the left/right projections on the cell and outputs a value which determines \( q \).
face_relaxationFloat This is the \( \omega _f \) from above. Chosen from \( ]0,1[ \).
approximate_system_matrix\( \mathbb{R}^{K^c (p+1)^d \times K^c (p+1)^d} \) Approximation \( \tilde{A} \) which will be internally inverted, so it has to be non-singular.
approximate_system_matrix_scaling\( \mathbb{R}^{K^c (p+1)^d \times K^c (p+1)^d} \)
integration_points[Float] Sequence of polynomial_degree+1 entries which identify the position of the integration points along the unit interval.
