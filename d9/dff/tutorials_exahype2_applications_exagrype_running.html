Use ExaGRyPE

The build and running of the ExaGRyPE is wrapped by a python interface for convenience. By calling the python script in command terminal, one can easily generate an executable that can run on any computational platform, from a student's laptop to supercomputer clusters. Different scenario can be picked by feeding different physics parameters into python script, the simulation setup and running parameter can also be controled directly via command line.

Users are referred to the general Tutorials of ExaHyPE 2 if they are interested in what is going on behind the screen. The script is also self-explaining in most parts.

Gauge Wave

The gauge wave sceranio represent a dynamic foliation of a static Minkowski spacetime, it serves a good benchmark setup for users (and also developers) to test their implemenations of the code. The detailed explanations and eqautions of this benchmark can be found in https://arxiv.org/abs/gr-qc/0305023, section IV.B.

To run a gauge wave simulation in ExaGRyPE, use the command

python3 ccz4.py -impl fd4-rk1-adaptive -s gauge -maxh 0.05 -minh 0.05 -ps 9 -plt 0.1 -et 0.5 -exn gauge --domain_r 0.5 --ReSwi 0 -cfl 0.1 --KOSigma 8.0
export OMP_NUM_THREADS=1
mpirun -n 1 ./peano4_test

Type parameter –help makes the Python script yield further info and an explanation of each parameter option. We recommend users not to change running parameters of ExaGRyPE during benchmarking as the parameter space is vast and some combinations lead to unpredictable behaviour.

Single Schwarzschild Black Hole

This scenario describes single stable, non-spinning black hole in the sapcetime, located in the center of the domain. One should expect the system becomes (quasi-) static after a lapsing time. The puncture method is adopted here to avoid the domain excision, but with a prize of high-resolution requirement.

The following example is computational-expensive thus not suitable for a normal PC:

python3 ccz4.py -ext adm -impl fd4-rk1-adaptive -s single-puncture -maxh 0.4 -minh 0.04 -ps 6 -plt 0.5 -et 100 -exn sbh --domain_r 9.0 --ReSwi 7 -cfl 0.1 -outdir /path/to/data/folder/ --KOSigma 8.0 -sommerfeld -so -interp second_order -restrict second_order

Binary Black Hole Merger

Two black hole rotates around each other in quasi-circular orbit, and finally merge at the center in this scenario. We set the initial physics condition of the system following https://arxiv.org/abs/gr-qc/0307027, Table I.

Again, the following example is not suitable for a normal PC:

python3 ccz4.py -impl fd4-rk1-adaptive -s two-punctures -maxh 0.4 -minh 0.04 -ps 8 -plt 0.5 -et 120 -exn bbh --domain_r 12.0 --ReSwi 6 -cfl 0.1 -outdir /path/to/data/folder/ --KOSigma 8.0 --BBHType 2 -sommerfeld -so -ext adm -interp second_order -restrict second_order

Clean up old and temporary files

The following statements remove all the glue code and autogenerated C++ files. That would be useful when you want to make sure no legacy code is around. There is no risk in deleting them - if you rerun the Python script, they will all be re-created.

rm *.o *.cmake
rm README-*.md
rm Makefile CMakeLists.txt
rm Abstract*
rm -rf celldata facedata globaldata observers tasks vertexdata repositories

To get rid of the output files, type in

rm *patch-file *vtu *.pvd *.bak *.csv

Visualisation

The output of ExaHyPE is given in peano patch file and one can use the following commands to convert them into normal vtu files, and inspect in other tools like paraview.

export PYTHONPATH=/path/to/Peano/python/
python3 /path/to/Peano/python/peano4/visualisation/render.py solution-CCZ4.peano-patch-file -j32
peano4
Definition CellMarker.h:14

The render script supports parallel snapshot processing and you can set the process number you would like to run in the same time by set parameter after -j.
