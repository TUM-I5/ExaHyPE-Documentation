266class SeparateSweepsWithEnclaveTasking(SeparateSweeps):
267 """!
268
269 Enclave variant of the solver where we still run through mesh once per Runge-Kutta sweep
270
271 The concept of (enclave) tasking within ExaHyPE solvers is described in
272 detail in the @ref page_exahype_solvers_enclave_solvers "generic enclave discussion of ExaHyPE".
273 This class is a prototype realisation of this concept which other solvers
274 then specialise for particular numerical schemes.
275
276 The class basically replaces the standard "update a cell" action set with an
277 action set that might or might not spawn a task. In return, it adds a further
278 action set which merges the arising task outcomes into the actual mesh
279 structure. By default, we use peano4::datamanagement::CellMarker::willBeEnclaveCell()
280 and peano4::datamanagement::CellMarker::hasBeenEnclaveCell() to guide the
281 decision whether to spawn a task or not. You can overwrite this decision
282 by redefining the corresponding entry in the dictionary befilled by
283 add_entries_to_text_replacement_dictionary().
284
285 ## Task priorities
286
287 Use the attributes self.enclave_task_priority to change the priority of the
288 task. This value can either be a string that C++ can evaluate into a
289 priority or a plain numerical value. I set it to
