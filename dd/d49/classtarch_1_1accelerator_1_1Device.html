Detailed Description

Device.

Any accelerator implementation has to provide a singleton Device. Its full qualified name is tarch::accelerator::Device. If no accelerator variant is switched on, Peano provides a default Device implementation that does nothing.

◆ getNumberOfDevices()

int tarch::accelerator::Device::getNumberOfDevices ( ) const

Return the number of GPUs that are available.

It is then Peano's policy that you access these GPUs with numbers from 0 to N. We work with logical GPU numbers, i.e. they might be mapped onto different numbers internally. How this mapping is realised however is strongly depending on the used accelerator implementation.
