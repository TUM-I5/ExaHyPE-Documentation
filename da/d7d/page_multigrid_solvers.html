There is also some important stuff in the abstract base class (from which all of them inherit from) - solvers.api.Solver.Solver.

There is a lot of docu about the DGCGCoupling to be found in solvers.api.actionsets.DGCGCoupling.AbstractDGCGCoupling The only one of these solvers actually in use is solvers.api.actionsets.DGCGCoupling.MultiplicativeDGCGCoupling, which works with the standard DGSolver (solvers.api.DiscontinuousGalerkinDiscretisationPointWiseRiemannSolver.DiscontinuousGalerkinDiscretisationPointWiseRiemannSolver), as well as the standard Collocated solver (solvers.api.CollocatedDLinearDiscretisationWithPointJacobi.CollocatedDLinearDiscretisationWithPointJacobi) if you want to use it in parallel, or the standard Multilevel CG solver single threaded (solvers.api.CollocatedMGSingleThreaded.CollocatedMGSingleThreaded) if single-threaded is fine too.
