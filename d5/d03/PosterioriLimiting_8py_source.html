39 """
40 All of the following are mostly used for limiting, they are set of but some classes that inherit from this
41 may turn them on.
42 Hold previous time step: if this is specified as true, the previous time step will be stored persistently.
43 The old value gets overwritten with the new value at the beginning of the predictor step.
44 This allows rollback of the solution to a previous state if for some reason the computations
45 yield incorrect results.
46 Used for limiting, where troubled cells are rolled back and recomputed with a more robust
47 scheme and neighbouring cells need their old solution to project values with.
48 PAC: if not none, a user-defined physical admissibility criterion should be checked at the end of
49 the correction step which verifies whether or not the produced solution is physically sensible.
50 number_of_pac_observables: number of variables which the physical admissibility criterion should consider

52 end of the corrector step. This essentially verifies whether shocks have formed.

90 @abstractmethod

92 """
93 This will serve as a face marker to communicate with neighbouring cells whether
94 a given cell is troubled, is neighbouring a troubled cell, or is fine.
95 Cells that are troubled or neighbouring a troubled cell must rollback their solution
96 to the previous one and perform one time step using a more robust FV scheme.
97 Therefore these and their neighbours must convert the DG representation of their
98 previous solution to an FV representation and project it to their faces so that these
99 can be exchanged with their neighbours.
100 """
101 Variants = ["REGULAR","REGULAR_TO_LIMITER","LIMITER_TO_REGULAR","TROUBLED"]
102 self._cell_label = exahype2.grid.create_cell_label(self._name)
103 self._face_label = exahype2.grid.create_face_label(self._name)
104 self._cell_label.data.add_attribute(dastgen2.attributes.Enumeration(name="Troubled_Marker", variants=Variants))

143 # We have to assume that the regular solver's time step size computation also considers troubled cells.
144 # Therefore, the minimal time step size of the regular solver is unnecessary small.

189 """
190 Saving cell needs to happen before the cell is used for any computations.
191 Checking troubledness needs to happen after any updates have been finalised.
192 Spreading limiter status is inconsequential, it can happen at any time since it has its own dedicated time step.
193 Copying and converting the data from regular to limiter can happen whenever as it
194 happens in a dedicated time step, it needs to be followed up by a projection of
195 said data, which in the case of an FV limiter is done automatically.
196 Copying from limiter to regular needs to happen after the limiter is finished
197 computing the new solution within a given cell.
198 """

200 self._action_set_check_troubledness.descend_invocation_order = 1000
201 self._action_set_spread_limiter_status.descend_invocation_order = 1
202 self._action_set_copy_convert_and_project_data.descend_invocation_order = 1000
