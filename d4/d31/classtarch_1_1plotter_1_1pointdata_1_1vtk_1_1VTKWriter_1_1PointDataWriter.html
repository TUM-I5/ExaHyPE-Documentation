If you close your writer, each point has to be assigned a value, i.e.

you may not add less data than you have points.

Implementation

The point writer (master class) does hold a writer for cells and one for vertices. This are used via plotPoints(). When we close the data writer, we first have to close both vertex and cell writer. For unstructured grids, we expect users to do this manually.

Here, we work with a lazy close mechanism. We check whether the vertices and cells are shutdown alerady. If not, we do it. Then we continue. This lazy implicit closing implies that we need another check in the master class, as there might situations where users dump particles without any data, i.e. this close() is never invoked.
