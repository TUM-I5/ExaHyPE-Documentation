Detailed Description

Introduce multiplicative CC Coupling.

The control logic here is simplistic and prescripted. We follow the following steps for the additive solver with one iteration:

  1. Pre First smoothing step
    • When we touchVertexFirstTime, we pass over rollOverAndPrepareCGVertex, which has the effect of taking whatever is in the CG solution and writing it into the Delta. This is then interpolated during touchCellFirstTime.
    • Interpolate correction, i.e. sum all the hierarchies up. This happens in touchCellFirstTime().
    • Suspend CG solver
    • Suspend DG solver
    • Let DG solver project data onto faces
  2. DG smoothing step
    • Suspend CG solver
    • Run DG solver
  3. Nth DG smoothing step
    • Still run DG solver (finish smoothing) During touchCellLastTime, we:
    • Compute hierarchical residual
    • Restrict
    • Suspend CG solver These steps will set newRestrictedRhs correctly
  4. Any CG solver On the first CG smoothing step, we should, during touchVertexFirstTime, pass over rollOverAndPrepareCGVertex which takes newRestrictedRhs and writes it into rhs, and we reset value to 0.
    • Suspend DG solver

Realisation

This action set will map onto a class of its own. We give the class a class attribute which we increment by one in each step. We then make the actual logic depend upon this counter, as we inject the predicates into the superclass: That is, the superclass has some boolean expressions which allow us to switch its features on and off. We make those guys depends upon the new counter.
