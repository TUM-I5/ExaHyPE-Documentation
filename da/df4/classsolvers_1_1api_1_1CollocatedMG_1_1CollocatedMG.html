Detailed Description

A full mathematical description will follow later, but for now we outline some algorithmic steps here.

We borrow heavily from the original Collocated Solver. See matrixfree_collocated_solver.

We intialise by setting the values for u and the RHS in implementation files. So far this has been tested with the standard poisson set up ( \( 8\pi^2 sin(x)sin(y) \) on the rhs, with zeroes in the initial guess). We do this only on the finest level, and every other level receives 0.

We enumerate each level according to its coarseness. Ie the top level is marked as 0, and we increase this number as we refine.

Algorithmic Blueprint

  1. On the finest level:
    • Solve with Jacobi for a pre-determined number of pre-smoothing steps. Use standard mass matrix
    • On the final sweep, we take the final residual and write this into the rhs of the vertex and one level above. We set its initial guess to 0.
    • We then move up one level coarser
  2. On the next level up:
    • Solve with Jacobi for a pre-determined number of pre-smoothing steps. Use identity instead of mass matrix
    • On the final sweep, we take the final residual and write this into the rhs of the vertex and one level above. We set its initial guess to 0.
    • We then move up one level coarser
  3. We continue until we are on the coarsest level. We then solve with Jacobi for the predetermined number of pre-smoothing steps.
  4. We then solve on the coarsest level with the predetermined number of post-smoothing steps.
    • After this, we move one level finer
  5. On the next level down:
    • When we touch a vertex for the first time, we add prolongation of the solution from the level above to our solution.
    • We then solve with the predetermined number of post-smoothing steps
    • After this, we move one level finer
  6. We continue until we reach the finest level, at which point we add prolongation from the level above, and run for predetermined number of post-smoothing steps. We then go back to step 1.

Every time we run a sweep on the finest level, we update the global residuals. We terminate when the solution on the finest level conforms to the given stopping criterion.

Implementation

We have 4 fundamental action sets, which we discuss here in order of invocation. Each of their operations is guarded by a bool function that lives in the abstract solver class.

UpdateSolution

See also
mghype.matrixfree.solvers.api.actionsets.CollocatedMGSolver.UpdateSolution

This is where we take the residual and diag, and set the solution on vertices to \( u + du = u + \omega / (diag * residual) \). We want to avoid zero-divsion, as reset the diagonal every sweep, meaning we need to wait until the second pre/post-smoothing sweep to update the solution. We also need to make sure we finalise the solution updates before we restrict/prolongate as well. updateSolution() in the abstract solver class handles all of this behaviour.

To make sure the solver works in multiple trees, we tell the vertices that lie on parallel domain boundaries to share residual and diagonal with each other, to make sure contributions from neighbouring cells are registered properly. See get_vertex_merge_method in this class.

This action set also makes sure that the RHS is correctly consolidated - we write our restrictions into the vertex field NewRhs. When we touch a vertex for the first time after restriction, we make sure that the rhs is set to the value of what we restricted (ie the newrhs).

Ditto for delta after prolongation.

Prolongation

See also
mghype.matrixfree.solvers.api.actionsets.CollocatedMGSolver.Prolongation

When we touch a vertex on the coarse level, we set the delta field to be \( U - OldU \). When we touch a vertex on the fine level, we prolongate the coarse delta into the fine level delta. On the next sweep, we update the fine solution to be \( u + delta \).

The abstract solver just needs to check if its smoothing state is set to prolongation, so the logic is much simpler.

ResetAndUpdateResidual

See also
mghype.matrixfree.solvers.api.actionsets.CollocatedMGSolver.ResetAndUpdateResidual

When we touch the vertex for the first time, we reset residual and diag to 0, or apply boundary conditions. When we touch the cell for the first time, we perform the jacobi iteration and write the residual into the \( 2^D \) adjacent vertices.

The abstract solver class does this if updateResidual() returns true, which it does when:

  • The refinement level matches the active level
  • The smoothing state is presmoothing or postsmoothing

Restriction

See also
mghype.matrixfree.solvers.api.actionsets.CollocatedMGSolver.Restriction

When we touch a coarse vertex for the first time, we reset everything to 0, i.e. rhs and solution. When we touch a fine vertex for the first time, we hand over to the mghype::matrixfree::solvers::cgmultigrid::restrictToNextLevel which writes the fine level residual into the newRhs of the level above.

Infrastructure

You can now specify the number of levels that you want to use in the solver itself. The finest level is set by the refinement of the mesh and is currently determined at runtime when the mesh is descended for the first time. If we find that the finest level isn't at least one larger than the coarest level, we throw a std::runtime_error.
