19class CollocatedMG(Solver):
20 """!
21
22 A full mathematical description will follow later, but for now we outline some algorithmic steps here.
23
24 We borrow heavily from the original Collocated Solver. See \ref matrixfree_collocated_solver.
25
26 We intialise by setting the values for u and the RHS in implementation files. So far this has been tested
27 with the standard poisson set up ( \f$ 8\pi^2 sin(x)sin(y) \f$ on the rhs, with zeroes in the initial guess).
28 We do this only on the finest level, and every other level receives 0.
29
30 We enumerate each level according to its coarseness. Ie the top level is marked as 0, and we increase this number
31 as we refine.
32
33 ## Algorithmic Blueprint
34
35 1. On the finest level:
36 - Solve with Jacobi for a pre-determined number of pre-smoothing steps. Use standard mass matrix
37 - On the final sweep, we take the final residual and write this into the rhs of the vertex and one level above. We set its initial guess to 0.
38 - We then move up one level coarser
39
40 2. On the next level up:
41 - Solve with Jacobi for a pre-determined number of pre-smoothing steps. Use identity instead of mass matrix
42 - On the final sweep, we take the final residual and write this into the rhs of the vertex and one level above. We set its initial guess to 0.
43 - We then move up one level coarser
44
45 3. We continue until we are on the coarsest level. We then solve with Jacobi for the predetermined number of pre-smoothing steps.
46
47 4. We then solve on the coarsest level with the predetermined number of post-smoothing steps.
48 - After this, we move one level finer
49
50 5. On the next level down:
51 - When we touch a vertex for the first time, we add prolongation of the solution from the level above to our solution.
52 - We then solve with the predetermined number of post-smoothing steps
53 - After this, we move one level finer
54
55 6. We continue until we reach the finest level, at which point we add prolongation from the level above, and run for predetermined number of post-smoothing steps. We then go back to step 1.
56
57 Every time we run a sweep on the finest level, we update the global residuals. We terminate when the solution on the
58 finest level conforms to the given stopping criterion.
59
60 ## Implementation
61
62 We have 4 fundamental action sets, which we discuss here in order of invocation. Each of their operations is guarded by a bool function that lives in the abstract solver class.
63
64 ### UpdateSolution
65
66 \see mghype.matrixfree.solvers.api.actionsets.CollocatedMGSolver.UpdateSolution
67
68 This is where we take the residual and diag, and set the solution on vertices to \f$ u + du = u + \omega / (diag * residual) \f$. We want to avoid
69 zero-divsion, as reset the diagonal every sweep, meaning we need to wait until the second pre/post-smoothing sweep to update the solution. We also
70 need to make sure we finalise the solution updates before we restrict/prolongate as well. updateSolution() in the abstract solver class handles all
71 of this behaviour.
72
73 To make sure the solver works in multiple trees, we tell the vertices that lie on parallel domain boundaries to share residual and diagonal with
74 each other, to make sure contributions from neighbouring cells are registered properly. See get_vertex_merge_method in this class.
75
76 This action set also makes sure that the RHS is correctly consolidated - we write our restrictions into the vertex field NewRhs. When we touch
77 a vertex for the first time after restriction, we make sure that the rhs is set to the value of what we restricted (ie the newrhs).
78
79 Ditto for delta after prolongation.
80
81 ### Prolongation
82 \see mghype.matrixfree.solvers.api.actionsets.CollocatedMGSolver.Prolongation
83
84 When we touch a vertex on the coarse level, we set the delta field to be \f$ U - OldU \f$. When we touch a vertex on the fine level, we prolongate
85 the coarse delta into the fine level delta. On the next sweep, we update the fine solution to be \f$ u + delta \f$.
86
87 The abstract solver just needs to check if its smoothing state is set to prolongation, so the logic is much simpler.
88
89 ### ResetAndUpdateResidual
90
91 \see mghype.matrixfree.solvers.api.actionsets.CollocatedMGSolver.ResetAndUpdateResidual
92
93 When we touch the vertex for the first time, we reset residual and diag to 0, or apply boundary conditions.
94 When we touch the cell for the first time, we perform the jacobi iteration and write the residual into the \f$ 2^D \f$ adjacent vertices.
95
96 The abstract solver class does this if updateResidual() returns true, which it does when:
97 - The refinement level matches the active level
98 - The smoothing state is presmoothing or postsmoothing
99
100 ### Restriction
101
102 \see mghype.matrixfree.solvers.api.actionsets.CollocatedMGSolver.Restriction
103
104 When we touch a coarse vertex for the first time, we reset everything to 0, i.e. rhs and solution.
105 When we touch a fine vertex for the first time, we hand over to the mghype::matrixfree::solvers::cgmultigrid::restrictToNextLevel
106 which writes the fine level residual into the newRhs of the level above.
107
108 ## Infrastructure
109
110 You can now specify the number of levels that you want to use in the solver itself. The finest level is set by the refinement of
111 the mesh and is currently determined at runtime when the mesh is descended for the first time. If we find that the finest level
112 isn't at least one larger than the coarest level, we throw a std::runtime_error.
113
114 """
